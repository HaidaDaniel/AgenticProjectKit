# Use Cases

## Greenfield start

Create a new repository with a clear workflow for AI-assisted development.

## MVP execution

Keep the project focused on a narrow set of tasks and avoid premature architecture.

## Product hardening

Move from prototype behavior to maintainable product behavior.

## Production readiness

Add safer workflow rules, more testing, and operational documentation.

## Brownfield adoption

Introduce kit structure into an existing repository without rewriting the codebase.

## Repository audit

Inspect an existing project and identify documentation, task, and instruction gaps.

## Multi-tool export

Generate consistent instructions for different AI coding tools from a single source.
